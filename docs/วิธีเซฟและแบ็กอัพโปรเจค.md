# วิธีเซฟและแบ็กอัพโปรเจค MTR (Check-in)

## 1. ใช้ Git + GitHub (แนะนำ)

### ขั้นตอนแรก – สร้าง Repo ในเครื่อง

เปิด PowerShell ในโฟลเดอร์โปรเจค (`mtr`) แล้วรัน:

```powershell
cd "c:\Users\yolo3\Downloads\mtr-20251105T112108Z-1-001\mtr"

# สร้าง Git repo
git init

# เพิ่มไฟล์ทั้งหมด (ไฟล์ใน .gitignore จะไม่ถูกเพิ่ม เช่น .env, __pycache__)
git add .

# commit ครั้งแรก
git commit -m "Initial commit - โปรเจค MTR Check-in"
```

### อัปโหลดขึ้น GitHub

1. ไปที่ [github.com](https://github.com) → สร้าง Repo ใหม่ (New repository)
2. **อย่า** เลือก "Add a README" ถ้าคุณมี README อยู่แล้ว
3. คัดลอก URL ของ repo (เช่น `https://github.com/username/mtr.git`)
4. ใน PowerShell รัน:

```powershell
git remote add origin https://github.com YOUR_USERNAME/YOUR_REPO.git
git branch -M main
git push -u origin main
```

หลังจากนี้ทุกครั้งที่แก้ไขแล้วต้องการเซฟ:

```powershell
git add .
git commit -m "อธิบายสิ่งที่เปลี่ยน"
git push
```

## 2. แบ็กอัพเป็นไฟล์ ZIP

- คลิกขวาที่โฟลเดอร์ `mtr` → **Send to** → **Compressed (zipped) folder**
- หรือใช้ PowerShell:

```powershell
Compress-Archive -Path "c:\Users\yolo3\Downloads\mtr-20251105T112108Z-1-001\mtr\*" -DestinationPath "c:\Users\yolo3\Desktop\mtr-backup-$(Get-Date -Format 'yyyyMMdd').zip"
```

ไฟล์ `.env` และ `__pycache__` อาจไม่ติดไปถ้าโดน .gitignore หรือการซ่อนไฟล์ — ถ้าต้องการให้ครบ ให้ copy โฟลเดอร์ทั้งหมดไปที่อื่นก่อนแล้วค่อย zip

## 3. แบ็กอัพฐานข้อมูล PostgreSQL

โปรเจคใช้ PostgreSQL (จาก `.env`: DB_NAME=checkin_db, DB_USER=postgres)

รันเมื่อ PostgreSQL เปิดอยู่:

```powershell
# ตรวจสอบว่า pg_dump อยู่ใน PATH (ถ้าติดตั้ง PostgreSQL แล้ว)
pg_dump -U postgres -h localhost -p 5432 checkin_db -F c -f "c:\Users\yolo3\Desktop\checkin_db_backup.dump"
# รหัสผ่านเมื่อถาม: 12345678 (ตามที่อยู่ใน .env)
```

หรือ export เป็น SQL ธรรมดา:

```powershell
pg_dump -U postgres -h localhost -p 5432 checkin_db > "c:\Users\yolo3\Desktop\checkin_db_backup.sql"
```

เก็บไฟล์ `.dump` หรือ `.sql` ไว้กับโปรเจคหรือที่ปลอดภัย จะได้กู้ข้อมูลกลับได้

## สรุปสั้น ๆ

| วิธี | ใช้เมื่อ |
|------|----------|
| **Git + GitHub** | ใช้ประจำ เพื่อเก็บประวัติและ sync หลายเครื่อง |
| **ZIP** | ต้องการสำรองทั้งโฟลเดอร์เป็นช่วง ๆ |
| **pg_dump** | ต้องการเก็บข้อมูลในฐานข้อมูล (รายชื่อ, คอร์ส, การเช็คชื่อ ฯลฯ) |

แนะนำ: ใช้ **Git + GitHub** เป็นหลัก และทำ **pg_dump** เป็นระยะถ้าข้อมูลใน DB สำคัญ
